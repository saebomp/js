Timeouts - Question 1
What is the order of the output of the letters and why?

console.log('a');

let timmy = setTimeout(function () { console.log('b') }, 1);

let timothy = setTimeout(function () { console.log('c') }, 10);

let timert = setTimeout(function () { console.log('d') }, 0);

console.log('e')

My Answer :

a -> e -> d -> b -> c

It will be excuted from top, but setTImeout will be excuted after the first and last console.log
console.log(d) will be third because it will display after 0, and console.log('b') will display after 10 milliseconds

Correct Answer :

a -> e -> b -> d -> c

It runs right at the very beginning and the second one is way down at the bottom. It's going to run the last one. We expect there to be some sort of delay with setTimeout.
It takes the setTimeout code off the main event loop and this will put aside the code and run at some point in the future after main event loop to be done.
console.log('b') will be in the secondary stack (this one have been put aside to run later) and it's the first one that's coming back to us. 1 milliseconds is too short, so it runs first before asynchronous area can determine the order.
Afterward, console.log('d') will run in 0 milliseconds. console.log('c') takes longer than everything else.
